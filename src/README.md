# 仓颉数据竞争静态检测工具

## 项目结构

```
src/
├── main.py              # 主入口程序
├── chir_parser/         # CHIR中间语言解析模块
│   ├── __init__.py
│   ├── parser.py        # CHIR解析器
│   └── ast_nodes.py     # AST节点定义
├── analyzer/            # 静态分析模块
│   ├── __init__.py
│   ├── concurrency.py   # 并发分析
│   ├── dataflow.py      # 数据流分析
│   └── race_detector.py # 数据竞争检测
├── output/              # 输出模块
│   ├── __init__.py
│   └── formatter.py     # 结果格式化
└── utils/               # 工具函数
    ├── __init__.py
    └── helpers.py
```

## 设计思路

### 1. CHIR解析
- 解析仓颉编译器生成的CHIR中间语言
- 提取并发相关结构（spawn, sync, mutex等）
- 构建控制流图(CFG)和调用图

### 2. 并发分析
- 识别spawn创建的线程
- 分析线程间的同步关系
- 构建线程逃逸分析

### 3. 数据竞争检测
- 共享变量识别
- 访问模式分析（读/写）
- 并发访问检测
- 同步保护分析

### 4. 输出格式
按照赛题要求的格式输出检测结果

## 数据竞争定义

根据仓颉语言规约，数据竞争发生在：
1. 两个或多个线程并发访问同一内存位置
2. 至少有一个是写操作
3. 没有适当的同步机制保护

## 检测类型
- RW: 读写竞争（一个线程读，另一个线程写）
- WW: 写写竞争（两个线程都写）
