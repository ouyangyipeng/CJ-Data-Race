# 仓颉数据竞争静态检测 - 项目进度记录

## 项目概述
- **赛题名称**: 仓颉数据竞争静态检测
- **比赛**: 2026全国大学生计算机系统能力大赛 编译系统设计挑战赛（华为毕昇杯）
- **目标**: 实现一个静态程序分析工具识别仓颉代码中潜在的数据竞争问题

## 关键技术要求

### 输入输出规范
- **输入**: 仓颉工程根目录地址字符串（可用cjpm编译构建）
- **输出**: .txt文件，每行一条数据竞争记录
- **分析层面**: CHIR（仓颉高级中间语言）

### 数据竞争类型
- **RW**: 读写竞争
- **WW**: 写写竞争

### 输出格式
1. 内部并发执行逻辑导致的数据竞争:
```
(RaceType，((filePath1,fileName1,spawnLine1),(filePath1',fileName1',raceLine1')),((filePath2,fileName2,spawnLine2),(filePath2',fileName2',raceLine2')))
```

2. 外部程序调用public接口导致的数据竞争:
```
(RaceType，((filePath1,fileName1,declareLine1),(filePath1',fileName1',raceLine1')),((filePath2,fileName2,declareLine2),(filePath2',fileName2',raceLine2')))
```

### 评分标准
- 50组测试用例，每组最高2分
- 时间限制: 5分钟内给出输出
- 得分公式: TP/(TP+FP)*1.0 + TP/(TP+FN)*1.0

## 技术要点记录

### 仓颉并发模型
- `spawn` 关键字创建新线程
- `Mutex` 互斥锁
- `RWLock` 读写锁
- `SpinLock` 自旋锁
- `Atomic` 原子操作
- `Channel` 通道

### 数据竞争定义
两个或多个线程并发访问同一内存位置，至少有一个是写操作，且没有适当的同步机制保护。

### 检测策略
1. 识别所有spawn创建的线程
2. 分析每个线程访问的变量
3. 检测跨线程的共享变量访问
4. 检查是否有同步保护
5. 报告潜在的数据竞争

### 关键实现
- 局部变量识别：基于作用域追踪
- 同步保护分析：识别lock/unlock配对
- 逃逸分析：标记被spawn捕获的变量
