# 仓颉数据竞争静态检测工具 - 设计文档

## 一、项目概述

### 1.1 项目背景
本项目为2026年全国大学生计算机系统能力大赛编译系统设计挑战赛（华为毕昇杯）参赛作品，旨在实现一个静态程序分析工具，用于识别仓颉代码中潜在的数据竞争问题。

### 1.2 项目目标
- ✅ 实现仓颉源码解析器，提取并发结构
- ✅ 实现数据竞争检测算法
- ✅ 输出符合赛题要求格式的检测结果
- ✅ 支持CHIR中间语言分析
- ✅ 支持多种同步原语（Mutex、RWLock、SpinLock、Atomic、Channel）

### 2.2 模块说明

#### 2.2.1 源码解析器 (CangjieParser)
- **功能**: 直接解析仓颉源代码(.cj文件)
- **输入**: 仓颉源文件或目录
- **输出**: 模块、函数、变量、spawn表达式等AST结构
- **关键特性**:
  - 识别spawn创建的线程
  - 识别变量声明和访问
  - 识别同步机制（Mutex、RWLock等）
  - 区分局部变量和共享变量

#### 2.2.2 CHIR解析器 (CHIRTextParser)
- **功能**: 解析仓颉编译器生成的CHIR中间语言文本格式
- **输入**: .chirtxt文件（由chir-dis工具生成）
- **输出**: CHIR模块、函数、基本块、表达式结构
- **关键特性**:
  - 解析Spawn表达式
  - 解析Load/Store内存访问
  - 解析函数调用
  - 提取源码位置信息

#### 2.2.3 增强并发分析器 (EnhancedConcurrencyAnalyzer)
- **功能**: 分析程序中的并发结构
- **支持的同步原语**:
  - Mutex（互斥锁）
  - RWLock（读写锁）
  - SpinLock（自旋锁）
  - Atomic（原子类型）
  - Channel（通道）
  - synchronized块
- **主要任务**:
  - 识别所有spawn创建的线程
  - 分析每个线程访问的变量
  - 识别同步区域和保护关系
  - 执行数据流分析
  - 标记逃逸变量

#### 2.2.4 数据竞争检测器 (RaceDetector)
- **功能**: 检测潜在的数据竞争
- **检测策略**:
  1. 线程间竞争检测
  2. Public接口竞争检测
  3. 同步保护分析
  4. 原子操作识别

#### 2.2.5 输出格式化器 (OutputFormatter)
- **功能**: 按赛题要求格式化输出
- **输出格式**:
  - 线程间竞争: `(RaceType,((filePath1,fileName1,spawnLine1),(filePath1',fileName1',raceLine1')),((filePath2,fileName2,spawnLine2),(filePath2',fileName2',raceLine2')))`
  - Public接口竞争: `(RaceType,((filePath1,fileName1,declareLine1),(filePath1',fileName1',raceLine1')),((filePath2,fileName2,declareLine2),(filePath2',fileName2',raceLine2')))`

## 三、数据竞争检测算法

### 3.1 数据竞争定义
根据仓颉语言规约，数据竞争发生在以下条件同时满足时：
1. 两个或多个线程并发访问同一内存位置
2. 至少有一个是写操作
3. 没有适当的同步机制保护

### 3.2 检测算法流程
```
输入: 仓颉工程目录
输出: 数据竞争列表

1. 解析源码，构建AST
2. 识别所有spawn创建的线程
3. 对每个线程，收集其访问的变量:
   - 记录读操作
   - 记录写操作
   - 记录同步保护状态
4. 检测线程间竞争:
   for each 线程对 (T1, T2):
       for each 共享变量 V:
           if (T1访问V && T2访问V):
               if (至少一个写操作):
                   if (没有同步保护):
                       报告数据竞争
5. 检测Public接口竞争:
   for each 公共函数对 (F1, F2):
       for each 共享变量 V:
           if (F1访问V && F2访问V):
               if (至少一个写操作):
                   报告潜在竞争
6. 格式化输出结果
```

### 3.3 同步保护分析
```
同步保护分析算法:
1. 识别同步区域:
   - synchronized块
   - lock/unlock调用
   - readLock/readUnlock调用
   - writeLock/writeUnlock调用

2. 建立保护关系:
   for each 同步区域 S:
       protected_vars = S内访问的变量
       for each 变量 V in protected_vars:
           V.protected_by.add(S.lock_object)

3. 竞争过滤:
   if (两个访问被同一锁对象保护):
       不报告竞争
```

## 五、使用方法

### 5.1 环境要求
- Python 3.8+
- 无需额外依赖

### 5.2 运行方式
```bash
# 方式1: 使用运行脚本
python3 run_detector.py <项目目录>

# 方式2: 作为模块使用
from src.chir_parser import CangjieParser
from src.analyzer import RaceDetector

parser = CangjieParser()
modules = parser.parse_directory("path/to/project")

detector = RaceDetector(modules)
races = detector.detect()
```

## 七、技术亮点

1. **双模式解析**: 支持源码直接解析和CHIR中间语言解析
2. **多同步原语支持**: 支持Mutex、RWLock、SpinLock、Atomic、Channel等
3. **数据流分析**: 实现了基本的数据流分析框架
4. **低误报率**: 通过同步保护分析和逃逸分析减少误报
5. **可扩展架构**: 模块化设计，易于扩展新的分析功能
