## 一、赛题内容详解

### 1.1 比赛背景

这是 **2026年全国大学生计算机系统能力大赛编译系统设计挑战赛（华为毕昇杯）** 的赛题之一。

**赛题名称**：仓颉数据竞争静态检测

**核心任务**：实现一个静态程序分析工具，识别仓颉（Cangjie）编程语言代码中潜在的数据竞争问题。

### 1.2 什么是数据竞争？

根据仓颉语言规约，数据竞争发生在以下条件**同时满足**时：

1. 两个或多个线程**并发访问**同一内存位置
2. 至少有一个是**写操作**
3. **没有适当的同步机制保护**（如Mutex、RWLock等）

**示例**：
```cangjie
// 存在数据竞争的代码
var counter: Int64 = 0

main() {
    spawn { counter = counter + 1 }  // 线程1写
    spawn { counter = counter + 1 }  // 线程2写
}
// 两个线程同时写counter，没有同步保护 → WW类型数据竞争
```

### 1.3 输入输出规范

#### 输入
- 一个表示仓颉工程**根目录地址**的字符串
- 目录下的仓颉工程满足可以被 `cjpm`（仓颉工程管理工具）编译构建

#### 输出
- 一个 `.txt` 文件
- 每行为一条识别出的数据竞争

#### 输出格式

**格式1：内部并发执行逻辑导致的数据竞争**
```
(RaceType，((filePath1,fileName1,spawnLine1),(filePath1',fileName1',raceLine1')),((filePath2,fileName2,spawnLine2),(filePath2',fileName2',raceLine2')))
```

- `RaceType`：竞争类型，取值为 `RW`（读写）或 `WW`（写写）
- 第一个元组：第一个线程的spawn位置和竞争位置
- 第二个元组：第二个线程的spawn位置和竞争位置

**格式2：外部程序调用public接口导致的数据竞争**
```
(RaceType，((filePath1,fileName1,declareLine1),(filePath1',fileName1',raceLine1')),((filePath2,fileName2,declareLine2),(filePath2',fileName2',raceLine2')))
```

- 第一个元组：第一个public接口的声明位置和竞争位置
- 第二个元组：第二个public接口的声明位置和竞争位置

### 1.4 技术要求

1. **必须在CHIR层面开发**：参赛队仅限于在仓颉高级中间语言CHIR上开发检测工具
2. **为开源编译器新增能力**：需要为开源的仓颉编译器或工具链新增数据竞争检测能力
3. **允许使用大模型**：可以本地部署或线上调用，但必须在文档中说明

## 二、赛题意义

### 2.1 实际应用价值

数据竞争是并发程序中最常见的bug来源之一：

- **Meta公司的RacerD**：已应用于Android工程级代码，累计帮助识别2500+问题
- **提前发现bug**：静态分析可在程序执行前识别问题，避免生产环境故障
- **降低调试成本**：并发bug往往难以复现和调试，静态检测可大幅降低成本

### 2.2 技术挑战

1. **精确性**：需要在减少误报（False Positive）和漏报（False Negative）之间平衡
2. **可扩展性**：需要处理大规模代码库
3. **语言特性**：仓颉语言有独特的并发模型和同步原语

### 2.3 学习价值

- 深入理解编译器原理和中间表示
- 掌握静态程序分析技术
- 学习并发理论和数据竞争检测算法

## 四、评分方式详解

### 4.1 初赛评分（满分100分）

#### 测试用例组成
- **50组测试用例**
  - 40组：拥有main函数入口的程序
  - 10组：允许被其他仓颉模块调用的包

#### 每组用例评分（最高2分）

**情况1：用例本身无数据竞争**
- 输出0条数据竞争 → 得2分
- 输出有数据竞争 → 不得分

**情况2：用例存在数据竞争**

得分公式：
```
得分 = TP/(TP+FP) × 1.0 + TP/(TP+FN) × 1.0
```

其中：
- **TP (True Positive)**：正确报告的真实数据竞争
- **FP (False Positive)**：误报的非数据竞争
- **FN (False Negative)**：漏报的真实数据竞争
- **TP+FP**：工具报告总数
- **TP+FN**：真实数据竞争总数

**时间限制**：
- 5分钟内完成：正常评分
- 3-5分钟：扣0.8分
- 超过5分钟：不得分

#### 评分示例

假设某测试用例有3个真实数据竞争：

| 情况 | TP | FP | FN | 得分计算 | 得分 |
|------|----|----|----|---------|-----|
| 完美检测 | 3 | 0 | 0 | 3/3×1 + 3/3×1 | 2.0 |
| 有漏报 | 2 | 0 | 1 | 2/2×1 + 2/3×1 | 1.67 |
| 有误报 | 3 | 1 | 0 | 3/4×1 + 3/3×1 | 1.75 |
| 全漏报 | 0 | 0 | 3 | 0/0×1 + 0/3×1 | 0 |
